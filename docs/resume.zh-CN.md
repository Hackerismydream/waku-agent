**Waku｜评测驱动的本地优先个人 Agent**

独立项目｜Python、SQLite FTS5、OpenTelemetry、Pytest、DeepEval、Docker、GitHub Actions｜2026 年 XX 月至今

面向长期运行的个人助手，独立实现多端接入、Agent Loop、Skill、长期记忆、可观测与 Eval 闭环。默认以本地 SQLite 保存用户状态，也可自托管至 ECS，通过 Web 与 Telegram 持续处理真实任务，并使用完整执行轨迹验证模型、Prompt 与 Skill 变更。

- 手写无框架 Agent 运行时，支持流式输出、多轮工具调用、最大迭代保护、会话恢复与高风险动作审批；抽象 Anthropic 和 OpenAI 两类协议适配层，统一接入 9 个模型提供方及 CLI、Web、Telegram、Voice 四类入口。
- 设计轻量模型控制层，负责请求分流、记忆检索、缺参判断与风险分类，复杂任务由主模型按需加载 Skill；基于 SQLite FTS5、检索门控和周期性巩固构建事实、情景、程序性三类记忆，沉淀 [Skill 数量] 个 Skill 与 [集成数量] 个外部集成，在留出集成功率不下降的前提下，使单任务主模型调用次数降低 [X]%，记忆相关留出任务成功率由 [A]% 提升至 [B]%。
- 建立执行轨迹级 Eval，人工复核 [黄金任务数] 条任务并划分开发集与留出集，使用确定性断言验证工具路径，以独立 LLM Judge 评估最终回复；每项任务在 [模型数量] 个模型上重复运行 [重复次数] 次，基于失败案例迭代 Prompt、路由与 Skill，使留出集端到端成功率由 [A]% 提升至 [B]%，在 CI 中拦截 [回归次数] 次行为退化。
- 将 Waku 部署至 ECS 持续运行，支持定时触发、持久化任务、待审批动作与中断恢复；通过 JSONL 和 OpenTelemetry 记录模型调用、工具执行、延迟与成本，在 [持续使用天数] 天内完成 [真实任务数] 个真实任务，P95 延迟为 [X] 秒，单个成功任务平均成本为 [X] 元。
